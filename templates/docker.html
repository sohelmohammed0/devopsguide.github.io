Docker & Containerization

Introduction to Docker

Docker is an open-source platform for automating the deployment, scaling, and management of applications in lightweight, portable containers. Docker containers encapsulate applications and their dependencies, making it easier to ensure consistency across development, testing, and production environments.

Key Benefits of Docker:

  • Improves portability by enabling applications to run in any environment.
  • Reduces conflicts between development, staging, and production environments.
  • Increases development speed through fast container startup and teardown.
  • Supports microservices architecture by allowing applications to be split into smaller, independently deployable services.

Best Practices for Docker Usage:

  • Keep images lightweight by minimizing layers and using small base images.
  • Use multi-stage builds to optimize image sizes.
  • Leverage Docker Compose for local development environments.
  • Ensure proper handling of sensitive data using environment variables or secrets.
  • Regularly clean up unused containers, images, and volumes to free up space.

Basic Docker Commands

Frequently Used Commands:

  • docker run <image> - Run a container from the specified image.
  • docker ps - List running containers.
  • docker images - List available Docker images.
  • docker build -t <tag> . - Build an image from a Dockerfile.
  • docker exec -it <container> /bin/bash - Access a running container's shell.

Advanced Commands:

  • docker-compose up - Start all services defined in a Docker Compose file.
  • docker swarm init - Initialize Docker Swarm for clustering and orchestration.
  • docker network ls - List all Docker networks.
  • docker volume ls - List all Docker volumes for persistent storage.
  • docker logs <container> - View the logs for a container (useful for debugging).

Sample Dockerfile for Practice

# Sample Dockerfile for a Node.js Application
FROM node:14-alpine

# Create app directory
WORKDIR /usr/src/app

# Install app dependencies
COPY package*.json ./
RUN npm install

# Copy application code
COPY . .

# Expose port and define startup command
EXPOSE 8080
CMD [ "node", "server.js" ]

Practice building an image from this Dockerfile by running docker build -t my-node-app . and then running the container with docker run -p 8080:8080 my-node-app.

Useful Code Snippets:

# Example Docker Compose YAML for Node.js and MongoDB
version: '3'
services:
  web:
    image: node:14-alpine
    volumes:
      - .:/usr/src/app
    ports:
      - "3000:3000"
    command: npm start
  db:
    image: mongo
    ports:
      - "27017:27017"

Use this Compose file to spin up a Node.js web service and a MongoDB database, then try connecting the web app to the database.
